HFS1 轨道交通控制系统 - 项目整体说明

1. 项目概述

1.1 项目简介

HFS1 是一套轨道交通列车自动监控系统（ATS）及其相关控制系统的完整软件实现，采用 C++ 开发，主要运行于 Linux 平台（部分模块支持 Windows）。系统采用分布式架构，由多个相对独立又紧密协作的模块组成，覆盖列车监控、调度、控制、通信、协议转换、人机界面等核心功能。

1.2 项目规模

指标 数值
总代码文件数（.cpp/.h/.hpp） 3,288 个
总代码行数 1,242,780 行
平均每文件行数 377 行
模块（目录）数量 100+ 个
MD 文档数量 75 份

1.3 系统架构特点

  • 分层架构：基础库 → 通信层 → 协议层 → 业务层 → 应用层
  • 模块化设计：每个目录对应一个功能模块，独立编译，松耦合
  • 分布式部署：多进程协同，基于消息通信
  • 高可靠性：1+N 主备架构、双网冗余、SIL2 安全完整性等级
  • 跨平台：主要支持 Linux（CENTOS），部分模块兼容 Windows

4. 关键模块详解

4.1 HmiTransInfo（HMI信息传输）

HmiTransInfo 模块负责将列车状态、报警、控制等信息转换为 MMI 终端可显示的绘制信息包。

  • HmiTransTask：完整HMI传输任务，发送所有元素（A1、A2、B、C1-C5等）
  • HmiTransQuickTask：快速HMI传输任务，只发送牵引制动状态（C1=33）和DTI（N=11）
  • 协议类型：TYPE_INFO_HMI_INDICATION = 0x66
  • 功能码：FUNC_TRAIN_ATP_HMI_ALL = 0x04

详见：HmiTransInfo模块说明

4.2 trcm（列车进路控制）

负责列车进路的计算、选排、控制和管理，支持路径搜索算法。

  • 核心类：CTrcm
  • 关键功能：进路选排、路径搜索、扣车、跳停、清客
  • 通信：与 Idec 系统的双向通信

详见：trcm模块说明

4.3 tocm（列车车载控制）

负责列车车载控制（ATO/ATP），与VOBC通信。

  • 核心类：CTocm
  • 关键功能：车载控制、移动授权、紧急制动

详见：tocm模块说明

4.4 ticm（列车综合控制）

负责列车综合控制，包括多种定时任务。

  • 核心类：CTicm
  • 关键定时器：基础定时器（1s）、AOM定时器（500ms）、滑行告警（3s）、门控（5s/10s）

详见：ticm模块说明

4.5 tda（调度自动化）

调度系统核心模块，处理调度计划和命令。

  • 核心类：CTda
  • 关键功能：调度计划管理、自动检查、计划分发

详见：tda模块说明

5. 核心业务信息流

5.1 跳停业务 

调度员操作 → OSM → tda → trcm（RouteCancelTask）
                              ↓
                          进路取消命令
                              ↓
                          车载（VOBC）→ tocm → ticm
                              ↓
                          跳停状态 → HmiTransInfo → MMI显示

5.2 清客业务 

调度员操作 → tda → trcm → 紧急制动原因第22位（制动重故障）
                              ↓
                          HmiTransInfo（C4区判断）→ MMI显示

5.3 扣车业务 

调度员操作 → tda → trcm（HoldTrainTask）→ 车载
                              ↓
                          HmiTransQuickTask → MMI显示

5.4 进路选排 

自动触发/人工操作 → trcm（AutoTriggerTask/RouteSelectTask）
                              ↓
                       PathSearch路径搜索
                              ↓
                       RoutesMgr进路管理
                              ↓
                       联锁（CBI）→ 进路执行

5.5 移动授权 

ticm → AtsZcProtocol → ZC（区域控制器）
                              ↓
                       移动授权计算
                              ↓
                       AtsVobcProtocol → VOBC → 列车控制

6. 通信机制

6.1 进程间通信（IPC）

  • 消息总线（msg）：基于 CMsgPack、CMsgHead 的消息封装
  • 业务网桥（brdg）：进程间业务消息桥接
  • 网络网桥（nbrdg）：跨节点网络消息桥接
  • 集群通信（cluster）：集群内多机通信

6.2 数据同步

  • Redis：作为共享数据存储，支持多进程共享状态
  • mssync：主备机数据实时同步
  • SyncProtocol：同步协议封装

6.3 外部接口

  • TCP/UDP：与外部系统通信
  • ODBC（db_wrap）：数据库访问
  • FTP（ftplib）：文件传输

7. 关键技术点

7.1 协议设计

  • 协议头：9字节，包含类型码、功能码、版本、包数、当前包、长度
  • 序列化/反序列化：每个协议类实现 ToStreamBuf 和 FromStreamBuf
  • 字节序：小端序（Little-Endian）

7.2 安全完整性（SIL2）

  • SIL2_code 目录：包含符合 SIL2 安全完整性等级的代码
  • 覆盖范围：AtsVobcProtocol、SpeedLimitProtocol、traininter、SpeedLimitProtocol
  • 关键特性：数据校验、冗余处理、故障检测

7.3 高可用设计

  • 1+N 主备架构：支持多机热备
  • 双网冗余：通信链路冗余
  • 故障检测：通过 nicmon、ifstate 监控通信状态
  • 状态恢复：通过 mssync 实现主备数据同步

8. 编译与构建

8.1 构建系统

  • Linux：基于 Makefile
  • Windows：支持 Visual Studio 工程
  • 构建脚本：script/build.sh

8.2 主要构建产物

  • 可执行文件：tda、ticm、tocm、trcm、tactcm、tdsm、cmsm 等
  • 动态库：libmsg、libbase、libRedisMgr 等
  • 配置文件：各模块的 .xml 配置文件

PublicExternals 模块功能说明

1. 模块概述

PublicExternals是系统的公共外部库模块，包含多个子模块，主要负责实现各种协议、插件公共库、网络通信库等核心功能。该模块为系统的其他模块提供基础支持，是系统的底层核心组件集合。

2. 主要功能

2.1 协议实现

  • 实现多种通信协议
  • 提供协议的编码和解码功能
  • 支持协议的注册和管理

2.2 插件公共库

  • 提供插件框架的公共接口
  • 实现插件管理器
  • 支持插件的加载、初始化和卸载

2.3 网络通信

  • 实现网络通信的基础功能
  • 支持客户端和服务器模式
  • 提供可靠的网络数据传输

2.4 配置管理

  • 实现系统配置的加载和管理
  • 支持多种配置文件格式
  • 提供配置的查询和更新功能

2.5 日志服务

  • 实现日志记录功能
  • 支持不同级别的日志
  • 提供日志文件的管理

3. 核心子模块

3.1 BaseProtocol

  • 基础协议库，实现协议的基础功能
  • 包含协议头、时间处理、引用计数等功能
  • 提供列车基础信息的实现

3.2 CtrlCmdProtocol

  • 控制命令协议库
  • 实现控制命令的编码和解码
  • 包含Vobc命令协议、Zc命令协议等

3.3 DagProtocol

  • DAG（数据采集网关）协议库
  • 实现DAG相关的协议功能

3.4 DiagramProtocol

  • 图形协议库
  • 实现图形相关的协议功能
  • 包含地图协议、通知协议等

3.5 DispatchProtocol

  • 调度协议库
  • 实现调度相关的协议功能
  • 包含IOC计划协议等

3.6 ESL

  • 日志服务库
  • 实现日志记录和管理功能
  • 支持不同级别的日志输出

3.7 HmiTransProtocol

  • HMI传输协议库
  • 实现HMI相关的协议功能

3.8 HmiViewPlug

  • HMI视图插件
  • 实现HMI视图的显示和管理
  • 包含图形元素、场景管理等功能

3.9 IndicationProtocol

  • 指示协议库
  • 实现指示相关的协议功能

3.10 InterfaceProtocol

  • 接口协议库
  • 实现不同系统之间的接口协议
  • 包含ISCS信息协议等

3.11 NetComm

  • 网络通信库
  • 实现网络通信的基础功能
  • 支持客户端和服务器模式
  • 包含TCP/IP通信、消息队列等功能

3.12 ParamMtnProtocol

  • 参数维护协议库
  • 实现参数维护相关的协议功能

3.13 PlugInCommon

  • 插件公共库
  • 实现插件框架的公共接口
  • 包含插件管理器、插件接口定义等

3.14 PublicConfig

  • 公共配置库
  • 实现系统配置的加载和管理
  • 包含线路配置、站点配置、设备配置等

4. 技术实现

4.1 模块化设计

  • 采用模块化设计，每个子模块负责特定功能
  • 模块之间通过接口进行通信
  • 便于扩展和维护

4.2 面向对象

  • 使用C++面向对象设计
  • 实现了丰富的类和接口
  • 支持继承、多态等面向对象特性

4.3 跨平台支持

  • 基于Qt框架实现
  • 支持Windows、Linux等多个平台
  • 使用CMake或qmake进行构建

4.4 插件架构

  • 实现了灵活的插件框架
  • 支持插件的动态加载和卸载
  • 提供插件的管理和生命周期管理

4.5 网络通信

  • 实现了可靠的网络通信
  • 支持TCP/IP协议
  • 实现了客户端和服务器模式

5. 与其他模块的关系

5.1 依赖关系

  • 其他模块依赖PublicExternals提供的基础功能
  • 插件模块依赖PlugInCommon提供的插件框架
  • 通信模块依赖NetComm提供的网络通信功能

5.2 调用关系

  • 其他模块通过PublicExternals提供的接口访问其功能
  • PublicExternals不直接调用其他模块

6. 关键文件说明

6.1 PlugInCommon

  • PlugInManager.h/cpp：插件管理器，负责插件的加载、初始化和卸载
  • PlugInInterfaceDef.h：插件接口定义，定义插件的标准接口
  • QATSMessageObject.h/cpp：消息对象，实现模块间的消息传递

6.2 NetComm

  • ClientBase.h/cpp：客户端基础类，实现客户端通信功能
  • ServerBase.h/cpp：服务器基础类，实现服务器通信功能
  • CommInterface.h/cpp：通信接口，定义通信的标准接口

6.3 ESL

  • ExLogService.h/cpp：扩展日志服务，实现日志记录功能
  • LogFile.h/cpp：日志文件类，负责日志文件的管理

6.4 BaseProtocol

  • BaseProtocol.h：基础协议定义
  • ProtocolHead.cpp/cpp：协议头处理
  • ProtocolTime.cpp/cpp：协议时间处理

7. 工作流程

7.1 插件加载流程

  1. 系统启动，初始化插件管理器
  2. 插件管理器加载插件配置文件
  3. 插件管理器加载所有插件
  4. 插件管理器初始化每个插件
  5. 插件向系统注册自己的功能

7.2 网络通信流程

  1. 初始化网络服务
  2. 创建客户端或服务器对象
  3. 建立网络连接
  4. 发送或接收数据
  5. 处理网络事件
  6. 关闭网络连接

7.3 日志记录流程

  1. 初始化日志服务
  2. 设置日志级别和输出方式
  3. 记录日志信息
  4. 管理日志文件

8. 技术特点

  • 模块化设计：采用模块化设计，便于扩展和维护
  • 跨平台支持：支持Windows、Linux等多个平台
  • 插件架构：实现了灵活的插件框架
  • 可靠的网络通信：提供可靠的网络数据传输
  • 全面的协议支持：支持多种通信协议
  • 完善的日志服务：提供全面的日志记录功能

9. 使用说明

9.1 模块引用

  • 其他模块通过包含相应的头文件引用PublicExternals的功能
  • 链接相应的库文件

9.2 配置文件

  • 各子模块使用自己的配置文件
  • 配置文件通常位于系统的配置目录

9.3 日志输出

  • 日志文件通常位于系统的日志目录
  • 日志级别可通过配置文件调整

10. 维护与扩展

10.1 功能扩展

  • 可添加新的协议实现
  • 可扩展网络通信功能
  • 可添加新的配置管理功能

10.2 性能优化

  • 优化网络通信性能
  • 优化协议处理速度
  • 优化配置加载效率

10.3 调试与测试

  • 可通过日志查看系统运行状态
  • 可使用测试工具验证功能

11. 子模块详细说明

11.1 PlugInCommon

  • PlugInManager：插件管理器，负责插件的加载、初始化和卸载
  • PlugInInterfaceDef：定义插件的标准接口
  • QATSMessageObject：实现模块间的消息传递

11.2 NetComm

  • ClientBase：客户端通信基础类
  • ServerBase：服务器通信基础类
  • CommInterface：通信接口定义

11.3 ESL

  • ExLogService：扩展日志服务
  • LogFile：日志文件管理

11.4 BaseProtocol

  • ProtocolHead：协议头处理
  • ProtocolTime：协议时间处理
  • TrainInfoBase：列车基础信息

11.5 PublicConfig

  • LayoutConf：布局配置管理
  • StationsConf：站点配置管理
  • LinesConf：线路配置管理

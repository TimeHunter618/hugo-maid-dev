## 1. 📖 项目概述

### 1.1 产品定位

WebSum 是一款基于大语言模型的 Chrome 浏览器侧边栏插件，帮助用户快速总结当前网页内容，提升信息获取效率。类似 Sider 的使用体验，但更轻量、更专注于网页摘要核心场景。

### 1.2 目标用户

| 用户画像 | 使用场景 |
|----------|----------|
| 🔬 **研究人员** | 快速了解论文、技术文章核心内容 |
| 🎓 **学生** | 总结课程资料、学习笔记 |
| 💼 **职场人士** | 快速浏览新闻、行业报告 |
| ✍️ **内容创作者** | 收集素材、提炼观点 |

### 1.3 核心价值

- ⚡ **高效获取信息**：一键生成网页摘要，30秒读完长文
- 🔍 **多维度理解**：支持要点提取、通俗解释、深度分析等多种模式
- 📱 **随时随地可用**：侧边栏常驻，不打断浏览流程
- 🔒 **数据隐私可控**：支持多种 LLM 提供商，API Key 本地加密存储

### 1.4 项目结构


## 2. ✨ 功能特性

### 2.1 核心功能

#### 2.1.1 📄 网页内容摘要

| 功能 | 说明 |
|------|------|
| 一键摘要 | 点击按钮立即生成当前网页的结构化摘要 |
| 多种模式 | 快速摘要、要点提取、通俗解释、深度分析 |

**摘要模式说明：**
- ⚡ **快速摘要**（默认）：生成 3-5 句话的精简摘要

- 💬 **通俗解释**：用简单语言解释专业内容，适合专业文章

#### 2.1.4 ⭐ 收藏管理

### 3.4 消息协议

扩展内部消息格式：
```javascript
{
  type: "MESSAGE_TYPE",
  payload: { ... },
  requestId: "uuid"
}
```

**核心消息类型：**

| 消息类型 | 方向 | 说明 |
|---------|------|------|
| `EXTRACT_CONTENT` | Panel → Content | 请求提取页面内容 |
| `EXTRACT_PAGE_CONTENT` | Panel → SW | 请求提取页面内容（代理） |
| `GENERATE_SUMMARY` | Panel → SW | 请求生成摘要 |
| `SUMMARY_CHUNK/DONE/ERROR` | SW → Panel | 摘要响应 |
| `CHAT` | Panel → SW | 发送聊天消息 |
| `CHAT_CHUNK/DONE/ERROR` | SW → Panel | 聊天响应 |
| `GET_AGENTS` | Panel → SW | 获取角色列表 |
| `RUN_DISCUSSION` | Panel → SW | 开始讨论 |
| `DISCUSSION_EVENT/DONE/ERROR` | SW → Panel | 讨论响应 |
| `GET_CONFIG/UPDATE_CONFIG` | Panel → SW | 配置管理 |
| `URL_CHANGED` | SW → Panel | URL 变化通知 |
| `TAB_ACTIVATED` | SW → Panel | 标签页切换通知 |

### 3.5 数据流

#### 3.5.1 摘要生成流程

```
用户点击「生成摘要」
        │
        ▼
Side Panel 发送 GENERATE_SUMMARY
        │
        ▼
Service Worker 调用 Content Script 提取网页正文
        │
        ▼
Service Worker 构建 Prompt，调用 LLM API（流式）
        │
        ▼
LLM 逐块返回 → SUMMARY_CHUNK → Side Panel 逐字显示
        │
        ▼
LLM 完成 → SUMMARY_DONE → 渲染 Markdown
```

#### 3.5.2 聊天流程

```
用户输入问题并发送
        │
        ▼
Side Panel 获取当前网页内容作为上下文
        │
        ▼
构建 messages（system 上下文 + 历史对话 + 用户问题）
        │
        ▼
Service Worker 调用 LLM API（流式）
        │
        ▼
LLM 逐块返回 → CHAT_CHUNK → Side Panel 打字机效果显示
        │
        ▼
LLM 完成 → CHAT_DONE → 渲染 Markdown
```

#### 3.5.3 讨论流程

```
用户选择角色并输入讨论主题
        │
        ▼
Side Panel 预渲染角色卡片（含 loading 动画）
        │
        ▼
Service Worker 发送 RUN_DISCUSSION
        │
        ├──▶ 尝试后端 API（如可用）
        │
        └──▶ 降级为本地 LLM 调用（逐个角色）
                │
                ▼
        每个角色流式返回 → DISCUSSION_EVENT(chunk)
                │
                ▼
        对应角色卡片逐字显示
                │
                ▼
        全部完成 → DISCUSSION_DONE
```

### 4.2 核心技术点

#### 4.2.1 🎯 内容提取双策略机制

**问题**：Content Script 可能因安全策略或加载顺序问题未注入

**方案**：
1. **策略一**：尝试 `chrome.tabs.sendMessage` 消息通信
2. **策略二**：回退到 `chrome.scripting.executeScript` 直接执行函数

```javascript
let result;
try {
  result = await chrome.tabs.sendMessage(tab.id, { type: 'EXTRACT_CONTENT' });
} catch (e) {
  const executeResult = await chrome.scripting.executeScript({
    target: { tabId: tab.id },
    func: extractPageContentFallback,
  });
  result = { success: true, data: executeResult[0]?.result };
}
```

#### 4.2.2 📊 正文提取算法

基于 DOM 节点密度分析：
- 标签语义识别（`<article>`, `<main>`, `<section>` 等）
- 文本密度计算（文本长度 / 标签数量）
- 常见选择器匹配（`.post-content`, `.article-body` 等）

#### 4.2.3 🔐 API Key 加密存储

| 项目 | 详情 |
|------|------|
| **算法** | AES-GCM 256位 |
| **密钥派生** | PBKDF2 + SHA-256，迭代 100,000 次 |
| **盐值** | 扩展 ID 派生的固定盐 |
| **存储** | 加密后的 Base64 字符串 |

## 5. � 核心问题与解决方案

### 5.1 内容提取失败

**现象**：`Could not establish connection. Receiving end does not exist`

**原因**：
1. Content Script 未注入
2. 消息监听器未注册
3. 跨域限制（CSP 策略）
4. 页面加载时机问题

**解决方案**：双策略提取机制（见 4.2.1）

### 5.2 讨论重复角色卡片

**现象**：每个角色显示两次

**原因**：两处都在创建角色卡片
1. `startDiscussion` 预渲染（带描述 + loading）
2. `round_start` 事件再次创建

**解决方案**：删除 `round_start` 中的重复创建逻辑

### 5.3 聊天打字机效果

**现象**：消息一次性出现，没有逐字显示

**原因**：
1. `requestId` 在 service worker 生成，前端监听器注册滞后
2. DOM 选择器错误（选中了 typing indicator 而非消息元素）

**解决方案**：
1. 前端生成 `requestId`，提前注册监听器
2. 使用 `data-stream-id` 属性精确定位消息元素

### 5.4 后端依赖降级

**设计理念**：确保无后端时所有功能可用

| 功能 | 后端可用 | 后端不可用 |
|------|----------|-----------|
| 网页摘要 | ✅ 直连 LLM | ✅ 不受影响 |
| 聊天对话 | ✅ 直连 LLM | ✅ 不受影响 |
| 角色列表 | 后端 .md（13个） | 本地 skills.js（18个） |
| 多角色讨论 | 后端并行调用 | 本地串行调用 LLM |
